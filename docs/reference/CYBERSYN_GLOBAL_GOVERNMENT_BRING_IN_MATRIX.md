## Validation — `REFERENCE.CATALOG` geography levels

Canonical **`geo_level_code`** values today live in `seeds/reference/catalog/geo_level.csv` 
(property, zip, county, cbsa, state, national, …, **`unmapped`**). Cybersyn **`LEVEL`** strings must still be 
crosswalked through **`REFERENCE.GEOGRAPHY.GEOGRAPHY_LEVEL_DICTIONARY`** (seed) so warehouse joins use 
stable codes such as **`zcta`**, **`tract`**, **`block_group`**, **`city`** — not raw vendor strings.
Until a new `LEVEL` is added to **`geo_level.source_snow_cybersyn_level`**, **`GEOGRAPHY_*`** models surface **`unmapped`** instead of NULL so dbt tests and joins stay strict.

| Cybersyn `LEVEL` (examples) | Dictionary `canonical_geo_level_code` |
|-----------------------------|----------------------------------------|
| CensusCoreBasedStatisticalArea | cbsa |
| CensusZipCodeTabulationArea | zcta |
| CensusTract | tract |
| CensusBlockGroup | block_group |
| County | county |
| State | state |
| City | city |

**Gap to close in catalog:** `geo_level.csv` uses **`zip`** for product grain; census-spine facts should prefer **`zcta`** where the geography is ZCTA-backed. Add or alias catalog rows when you register Cybersyn-backed datasets so `GEO_LEVEL_CODE` on facts lines up with both the dictionary and `REFERENCE.CATALOG.geo_level`.

## Naming — **latest** vs source `_pit` tables

Design vocabulary uses **latest** (and **as-of**) for current-row and time-scoped reads. When a **source** `table_name` ends with `_pit`, treat it as **latest / as-of / history-grain** support in **SOURCE_SNOW** only—not a label for REFERENCE models. Do not use the term **PIT** in prose unless you are explicitly citing the `_pit` substring in a Snowflake object name.

## Placement rules

- **REFERENCE** — canonical shared utilities and reusable spines (dictionary, index, codes, shapes, relationships, flattened current join).
- **SOURCE_SNOW** — native share tables; read or thin-wrap; do not dump raw vendor copies into REFERENCE.
- **TRANSFORM** — normalized tall `FACT_*` / `CONCEPT_*` built from SOURCE_SNOW + REFERENCE joins, QA, semantic-ready.

This catalog lists **SOURCE_SNOW** base `table_name` values only. **`TRANSFORM`** objects are your dbt models (not rows in `CYBERSYN_DATA_CATALOG`).

## Matrix — `tier` column (priority tiers)

| tier | Meaning |
|------|---------|
| **1** | Tier 1 — geography backbone |
| **2** | Tier 2 — housing, demographics, migration core |
| **3** | Tier 3 — housing finance and mortgage performance |
| **4** | Tier 4 — financial institutions and lender footprint |
| **5** | Tier 5 — crime, weather, disaster, risk overlays |
| **6** | Tier 6 — address and POI support layers |
| **88** | Supporting macro (e.g. BLS, Fed) — not first-order housing price series |
| **99** | Other catalog objects — **de-prioritized** for first-order geography + housing intelligence unless a product explicitly needs them (e.g. `openalex_*`, `github_*`, most `sec_*`, `world_bank_*`, `oecd_*`, other broad international macro) |

## Recommended first-cut shortlist (minimum viable geography + housing)

| table_name | why |
|------------|-----|
| `geography_index` | Registry for every downstream geo join and LEVEL crosswalk. |
| `geography_characteristics` | FIPS, WKT, GeoJSON payloads for codes + shapes. |
| `geography_relationships` | CBSA/county/tract/ZCTA rollups and overlaps without bespoke SQL. |
| `geography_hierarchy` | Single-parent paths when relationships are too permissive. |
| `geography_overlaps` | Tract↔ZCTA-style allocation edges for housing market ZIP/ZCTA semantics. |
| `geography_*` tables whose names end with `_pit` | Latest/as-of/history-grain companion reads when warehouse rows must match Cybersyn vintage keyed off `_pit` source tables. |
| `us_real_estate_attributes` + `us_real_estate_timeseries` | Core Cybersyn real-estate metrics spine. |
| `housing_urban_development_attributes` + `housing_urban_development_timeseries` | HUD-aligned housing supply/demand context. |
| `american_community_survey_attributes` + `american_community_survey_timeseries` | Demographic / tenure / rent burden controls at census geographies. |
| `irs_migration_by_characteristic_timeseries` + `irs_origin_destination_migration_timeseries` | Migration demand-side narratives tied to counties/states. |
| `usps_address_change_timeseries` | Mobility / churn proxy at postal-ish geographies. |

**Why this set:** you cannot build a governed **TRANSFORM** fact layer on Cybersyn housing series without the **geography backbone** (index + characteristics + relationships + hierarchy + overlaps) and the **variable + timeseries** pairs for real estate, HUD, ACS, and migration. Everything else in the catalog either supports overlays later (FHFA, HMDA, crime, weather, FEMA) or is intentionally de-prioritized for first-order real-estate intelligence.
